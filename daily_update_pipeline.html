Power Query — Daily Update

โฟลเดอร์ต้นทางกับไฟล์ report แยกกันคนละที่ แต่เชื่อมกันผ่าน query

แผนภาพนี้แสดงว่าไฟล์ไหนอยู่ที่ไหน, ลำดับการประมวลผลจริงในตัว query, และจุดที่ทั้งสองฝั่งเชื่อมกัน

Source Folder — ที่ไหนก็ได้ ตั้งชื่อเองได้

Daily_Update_2567.xlsm sheet "Daily Update"
... ไฟล์อื่น ๆ ในโฟลเดอร์
~$Daily_Update_xxx.xlsm lock file — ถูกกรองทิ้ง
อ่านผ่าน FolderPath ที่ตั้งไว้ใน ConfigTable (sheet "config" ในไฟล์ report) — ย้ายโฟลเดอร์เมื่อไหร่ แก้ค่าใน sheet แล้ว Refresh ไม่ต้องเปิด M code
ชื่อไฟล์ต้องลงท้ายด้วยเลขปี 4 หลักเสมอ (เช่น ..._2566) — query ดึงปีจาก 4 ตัวอักษรสุดท้ายก่อนนามสกุลไฟล์ตรงๆ ไม่ได้ค้นหารูปแบบปี
Folder.Files(FolderPath) สแกนหาไฟล์ .xlsm ทั้งหมด (ยกเว้น lock file ~$ ที่โผล่ตอนมีคนเปิดไฟล์ค้าง)

query ที่ทำงานนี้: Daily_Update_Combined.pq
ผลลัพธ์
โหลดเข้า sheet ผลลัพธ์ในไฟล์ report เดียวกันนี้ (Close & Load) — มีคอลัมน์ SourceFile และ ปี ติดมาด้วย
Refresh = ดึงข้อมูลใหม่จากทุกไฟล์อีกครั้ง
1

ไฟล์ report กับโฟลเดอร์ต้นทางไม่ต้องอยู่ที่เดียวกัน และไฟล์ report ตั้งชื่ออะไรก็ได้

Excel.CurrentWorkbook() อ้างถึง "workbook ที่ query กำลังรันอยู่" เท่านั้น ไม่ผูกกับชื่อไฟล์หรือ path เลย

2

SelectColumnTable ต้องอยู่ในไฟล์ report เดียวกันกับ query เสมอ — ข้าม workbook ไม่ได้

ถ้าเปลี่ยนชื่อ Table เป็นอย่างอื่น query จะหาไม่เจอและ error ทันที

3

ลำดับจริงคือ clean header → เลือกคอลัมน์ → รวมไฟล์ ไม่ใช่รวมก่อนแล้วค่อยเลือกคอลัมน์

กรองคอลัมน์ทิ้งตั้งแต่ระดับไฟล์ก่อน ทำให้ตอน combine เบากว่า ไม่ต้อง union คอลัมน์ที่ไม่ใช้จากทุกไฟล์

4

ถ้าโฟลเดอร์ไม่มีไฟล์ที่ใช้ได้เลย query จะ error พร้อมข้อความชัดเจนบอก FolderPath ที่ใช้อยู่

เจอข้อความนี้ = path ผิด หรือไม่มีไฟล์ไหนมี sheet "Daily Update" — ไม่ใช่ query พัง

5

คอลัมน์ ปี มาจาก 4 ตัวอักษรสุดท้ายของชื่อไฟล์ก่อนนามสกุลตรงๆ — ไม่ใช่การค้นหารูปแบบปีในชื่อ

ถ้าชื่อไฟล์ไม่ได้ลงท้ายด้วยเลขปี 4 หลัก (เช่นลงท้ายด้วยวันที่) จะได้ null แทน ไม่ error แต่ไม่มีปีให้ใช้

6

การตั้งค่าทั้งหมดแก้จาก sheet "config" (ConfigTable) แล้วกด Refresh — ไม่ต้องเปิด M code อีก

query ที่ต้องโหลดในไฟล์ report มี 4 ตัว: fnCleanMergedHeaders, fnCleanMergedHeadersSkipJunk, Config, Daily_Update_Combined (+ Daily_Update_HeaderList ถ้าใช้ตัวช่วยดู header)
